submission

Submission process shortly:

1. Participants run their system with evaluation dataset, and produce the system output in specified format.

2. 参赛者需要整理系统代码。完整的代码需要实现读取评估数据集，计算，生成系统输出的功能。

3. Participants describe their system in a technical report in sufficient detail.

4. Participants prepare the submission package (zip-file). The submission package contains system outputs, a complete system code that can be run (in a separate folder) and the technical report.

Submission system

提交系统将与2020年8月15日，评估集数据发布后开放。

By submitting to the challenge, participants agree for the system output to be evaluated and to be published together with the results and the technical report on the Challenge website under CC-BY license.

Submission file label

To avoid overlapping labels among all submitted systems, use following way to form your label:

[first name]_[last name]_[Abbreviation of institute of the corresponding author]

For example the systems would have the following labels:

• Hua_Li_BUPT

• Michael_Jordan_UNC

Make sure your zip-package follows provided file naming convention and directory structure. The zip package example can be downloaded here.

                        Hua_Li_BUPT.zip                                Zip-package root, Task submissions
                        │
                        │   Hua_Li_BUPT_technical_report.pdf           Technical report
                        │   Hua_Li_BUPT_output.csv                     Task System output
                        │
                        └───Hua_Li_BUPT_code                           Task System code
                                  code.py

System outputs

Participants must submit the results for the provided evaluation datasets.

System output should be presented as a single text-file (in CSV format, with a header row) containing classification result for each audio file in the evaluation set. The following system output format must be followed.

                        file name               style               Human                  AI

                        bach-0.mid               bach                0.8                   0.2
                        bach-1.mid               bach                0.75                  0.25
                        pop-0.mid                pop                 0.6                   0.4
                        pop-1.mid                pop                 0.1                   0.9

System code

参赛者需要提交比赛使用的完整代码，代码需要实现读取评估数据集，计算，生成系统输出的功能，并且说明使用到的第三方库。

Technical report

All participants are expected to submit a technical report about the submitted system, to help the organizer better understand how the algorithm works. Technical reports can only be used in Chinese or English.
