人 工 智 能 作 曲
识 别 比 赛

人工智能作曲
逐步兴起，越来越多的人工智能作曲技术被引入商业领域应用， 由此也引发了一系列潜在的知识产权纠纷以及关于知识产权侵权行为认定标准及事实认定程序的思考， 比如：对于人工智能作曲算法的盗用，甚至将乐曲旋律的人为侵权行为归结为人工智能作曲系统的算法缺陷， 自然人的主观判断的客观程度，诉讼程序对专业鉴定的依赖程度等。为了应对这些潜在的知识产权挑战， 我们需要一种客观化的音乐旋律描述方法和判断标准，以在出现相关知识产权纠纷时， 完成有关人工智能自动作曲系统的数字取证工作，并完成侵权行为及其程度的判断与认定工作。

距 离 发 布 训 练 数 据 剩 余

数据赛介绍

本次挑战赛的目的是对人工智能生成的单旋律音乐进行识别，即判别特定旋律是否由人工智能算法生成。 比赛使用两种曲风有明显区别的数据库，分别对几种工作原理有明显区别的人工神经网络自动作曲系统进行训练， 而后根据得到的模型生成一定数量人工智能音乐旋律， 音乐为MIDI格式，参赛者需要使用算法对音乐进行分类，即是人工智能生成的音乐， 还是人类创作的音乐。比赛以分类准确率作为评判标准。

训练数据（用作训练自动作曲生成系统）

本次比赛使用两种风格的音乐分别训练模型，两种风格为流行音乐及巴洛克时期Bach的音乐。 其中流行音乐只使用旋律部分，Bach的音乐将每一声部的旋律作为单旋律。

开发数据集

开发数据集包含了6000个MIDI文件，每个文件为单旋律音乐，曲速为120bpm， 每首旋律长度为8小节，由不同的生成系统得到。 对于人类的音乐，随机在两种风格的数据库中挑选一部分旋律，并切分8小节长度（该8小节并不一定为完整乐句）， 曲速调整为120bpm，人类音乐数据部分。 其中人工智能旋律与原有数据库中旋律有对应的标签，注意数据集中也会对不同风格的音乐进行标注， 但此任务不需要对风格进行分类。 开发数据包含6000首MIDI文件及一个csv文件，MIDI文件以下列方式命名。 

[composer label]-[style]-[id].mid

csv文件给出了对应文件的标签如下表所示，composer label中“0”对应的是人工智能作曲，“1”对应的是人类作曲。

file name style composer label
0-bach-0.mid bach 0
0-bach-1.mid bach 0
1-pop-0.mid pop 1
1-pop-1.mid pop 1
开发集中的数据数量如下表所示。

开发数据集
4000首MIDI（8小节长）

评估数据集

评估数据集包含了2000个MIDI文件，每个文件为单旋律音乐，曲速为120bpm， 每首旋律长度为8小节。 对于人工智能生成的音乐，旋律来源同开发集一样。 对于人类的音乐，在评估集中人类的音乐有两种来源，其中90%的音乐以和开发集相同的处理方法得到， 并且不包含开发集中已选择的音乐不同。 另外10%的音乐，我们从作曲家目前尚未发表的作品中得到，这部分音乐不存在于目前任何已知的数据库当中。 评估集中的MIDI文件以下面的方式命名。 

[style]-[id].mid

提交

Official challenge submission consists of:

• 系统的输出文件（*.csv）

• 对系统作出详细解释的技术报告（*.pdf）

• 可以运行的完整系统（放在单独的文件夹中）

系统输出文件应为单个文件（CSV格式，带有标题行），其中包含评估数据集中 每一个MIDI文件的分类结果，并且结果需要包含每个标签的输出概率值。

All files should be packaged into a zip file for submission.

提交的详细信息请查看提交页面。

外源数据

本次比赛可以使用外源数据，但必须符合以下要求。

• 外源数据必须为开源数据，并且可以被引用。外源数据可以是公开的数据集或者预训练好的模型，这些数据必须在2020年8月15日前公开并免费提供

• 如果使用了公开的数据集作为外源数据，则参赛者须提交至少一份未使用外源数据的系统，这样我们可以探讨外源数据对比赛的影响。 如果使用了预训练好的模型等非音乐数据作为外源数据，则无需遵守以上要求。

• 参赛者需要在技术报告中将使用过的外源数据详细列出。

• 参赛者如果使用了公开的数据集，在比赛过程中需要及时通知主办方，主办方将在网站上发布公开数据集信息， 使得所有参赛队伍都可以同等地使用该数据集。当测试数据集发布时，公开数据集列表将被锁定，之后参赛者禁止使用其他公开的数据集。

比赛规则

参赛者必须遵守以下规则。

• 参赛者允许使用外源数据，具体使用条件参照外源数据部分

。

• 参赛者可以对提供的数据进行任何操作。

• 参与者禁止对评价数据进行主观判断和标注。评估数据集不能用来训练提交的系统

• 分类决策必须对每个测试样本独立进行。
