流程

1. 参赛者使用评估数据集运行他们的系统，并以指定的格式生成系统输出。

2. 参赛者需要整理系统代码。完整的代码需要实现读取评估数据集，计算，生成系统输出的功能。

3. 参赛者在技术报告中详细介绍其识别算法，并提交一份技术报告。

4. 参赛者将三种文件打包成zip，并提交。

提交系统

提交系统将与2020年8月15日，评估集数据发布后开放。

参与者同意对系统输出进行评估，并在CC-BY许可下将结果和技术报告一起发布在人工智能作曲识别比赛网站上。

提交文件命名

为了避免参赛者提交的文件重名，请使用以下规则来命名提交的文件。

[first name]_[last name]_[Abbreviation of institute of the corresponding author]

如下面的例子所示。

• Hua_Li_BUPT

• Michael_Jordan_UNC

确保提交的zip文件遵循以下的目录结构。zip包示例可以这里下载。

                        Hua_Li_BUPT.zip                                Zip-package root, Task submissions
                        │
                        │   Hua_Li_BUPT_technical_report.pdf           Technical report
                        │   Hua_Li_BUPT_output.csv                     Task System output
                        │
                        └───Hua_Li_BUPT_code                           Task System code
                                  code.py

系统的输出文件

参赛者必须为提供的评估数据集提交结果。

系统的输出文件为CSV格式，且带有标题行。输出结果需要包含人工智能音乐及人类音乐类别的输出概率（浮点数类型）， 以及两种风格的标签（字符串类型）， 如下表所示。

                        file name               style（str）               Human（float）                  AI（float）

                        0.mid                   A                          0.8                            0.2
                        1.mid                   A                          0.75                           0.25
                        2.mid                   B                          0.6                            0.4
                        3.mid                   B                          0.1                            0.9

系统代码

参赛者需要提交比赛使用的完整代码，代码需要实现读取评估数据集，计算，生成系统输出的功能，并且说明使用到的第三方库。

技术报告

参赛者需提交有关提交系统的技术报告，以帮助主办方更好地了解该算法的工作原理。技术报告只能使用中文或者英文两种语言。
