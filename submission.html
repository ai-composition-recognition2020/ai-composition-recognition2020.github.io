数据赛介绍

本次挑战赛的目标是对人工智能生成的单旋律音乐进行识别。 项目使用两种曲风有明显区别的数据库，分别对三种工作原理有明显区别的人工神经网络自动作曲系统进行训练， 而后根据得到的六种模型生成一定数量人工智能音乐旋律， 音乐为MIDI格式，具体识别任务可以分为两个不同的子任务。

数据集

本次比赛使用两种风格的音乐分别训练模型，两种风格为流行音乐及巴洛克时期Bach的音乐

音乐生成算法

• 系统A MusicTransformer： 论文链接 该系统基于神经网络语言模型，用自回归的方式，即根据前面的音符推测后面的音符， MusicTransformer使用了transformer作为特征提取器，可以学习到音符之间长距离依赖关系。

• 系统B MusicVAE 论文链接 该系统基于变分自编码器（Variational Autoencoder, VAE）实现。现有的VAE模型很难对具有长期结构的序列进行建模。 为了解决这个问题，MusicVAE建议使用一个分层解码器，它首先输出输入子序列的嵌入，然后使用这些嵌入独立地生成每个子序列。

• 系统C MidiNet 论文链接 该系统基于生成对抗网络（Generative Adversarial Networks, GAN）实现。 生成器和判别器均使用了卷积神经网络（Convolutional Neural Network, CNN）作为特征提取器，使用生成对抗的方式进行训练。 生成器每次由高斯噪声生成一个小节的音乐，并且之前小节的音乐信息会作为控制信息加入，以此来保证音乐的长距离依赖性。

子任务 A：判别特定旋律是否由人工智能算法生成

该任务的目的是辨别人工智能生成的旋律与人类创作的旋律，数据集中包含了两种风格的旋律， 流行音乐及巴洛克时期巴赫的音乐，任何一种风格的音乐都包含人工智能生成的及人类创作的， 参赛者需要将人工智能生成的旋律及人类创作的旋律进行分类， 比赛以分类准确率作为评判标准。

开发数据集

开发数据集包含了4000个MIDI文件，每个文件为单旋律音乐，曲速为120bpm， 每首旋律长度为8小节，其中2000首为人工智能算法生成，2000首来自原有数据库。 对于人工智能生成的音乐，使用流行音乐和巴赫音乐的数据库，分别对三种自动作曲系统进行训练得到六种模型。 使用六种模型各生成一定数量的旋律组成2000首。 对于人类的音乐，随机在两种风格的数据库中挑选一部分旋律，并切分8小节长度（该8小节并不一定为完整乐句）， 曲速调整为120bpm，组成2000首。 其中人工智能旋律与原有数据库中旋律有对应的标签，注意数据集中也会对不同风格的音乐进行标注， 但此任务不需要对风格进行分类。 开发数据包含2000首MIDI文件及一个csv文件，MIDI文件以下列方式命名。 


                    [composer label]-[style]-[id].mid
csv文件给出了对应文件的标签如下表所示，composer label中“0”对应的为人工智能旋律，“1”对应的为人创作的旋律； style label中“0”对应的为流行音乐，“1”对应的为巴赫音乐。

file name composer label style label
xxx.mid 0 0
xxx.mid 1 1

评估数据集

评估数据集包含了2000个MIDI文件，每个文件为单旋律音乐，曲速为120bpm， 每首旋律长度为8小节。 对于人工智能生成的音乐，旋律来源同开发集一样，有1000首。 对于人类的音乐，在评估集中人类的音乐有两种来源，其中90%的音乐以和开发集相同的处理方法得到， 并且不包含开发集中已选择的音乐不同。 另外10%的音乐，我们从作曲家目前尚未发表的作品中得到，这部分音乐不存在于目前任何已知的数据库当中。 评估集中的MIDI文件以下面的方式命名。

子任务 B：判别人工智能算法生成的旋律属于哪种生成算法

该任务的目标是辨别人工智能生成的旋律属于三种自动作曲系统中的哪一种， 将三种系统生成的两种风格的旋律混合在一起，每一个系统生成的旋律有相应的标签。 参赛者需要将旋律分成三类，每一类对应一个自动作曲系统。比赛以分类准确率作为评判标准。

开发数据集

开发数据集包含了3000个MIDI文件，均为人工智能生成。每个文件为单旋律音乐，曲速为120bpm， 每首旋律长度为8小节，其中1000首为系统A生成；1000首为系统B生成；1000首为系统C生成 不同系统生成的旋律标记有对应的标签。 开发数据包含3000首MIDI文件及一个csv文件，MIDI文件以下列方式命名。 


                    [system label]-[style]-[id].mid
csv文件给出了对应文件的标签如下表所示， system label中“A”对应的为生成系统A，“B”对应的为生成系统B，“C”对应的为生成系统C； style label中“0”对应的为流行音乐，“1”对应的为巴赫音乐。
