Introduction

The purpose of this challenge is to identify single melody music generated by artificial intelligence, that is, to determine whether a specific melody is generated by an artificial intelligence algorithm. The competition uses two databases with distinct styles to train three different artificial neural network automatic composition systems with distinct working principles, and then generates a certain amount of artificial intelligence music melody according to the obtained six models. The music is in MIDI format , Contestants need to use algorithms to classify music from different sources. The competition uses the classification accuracy as the judging criterion.

Training Dataset (for generation system)

This competition uses two styles of music to train the model separately. The two styles are pop music and Bach music in the Baroque period. Among the popular music, only the melody part is used, and Bach's music uses the melody of each voice as a single melody.

Development dataset

The development data set contains 4000 MIDI files, each of which is single-melody music, the tempo is 120bpm, and each melody is 8 bars long, of which 2,000 are generated by artificial intelligence algorithms and 2,000 are from the original database. For the music generated by artificial intelligence, using the databases of popular music and Bach music, the three automatic composition systems were trained to obtain six models. Use six models to generate a certain number of melodies to form 2000 songs. For human music, randomly select a part of the melody in the database of two styles, and divide the length of 8 measures (the 8 measures are not necessarily complete phrases), and the tempo is adjusted to 120 bpm to form 2000 songs. Among them, the melody of the artificial intelligence has a corresponding label to the melody in the original database. Note that the music set will also label different styles of music, but this task does not need to classify the style. The development data includes 2000 MIDI files and a csv file. The MIDI files are named in the following manner. 

[composer label]-[style]-[id].mid

The csv file gives the label of the corresponding file as shown in the following table. "0" in the composer label corresponds to artificial intelligence composition, and "1" corresponds to human composition.

file name style composer label
0-bach-0.mid bach 0
0-bach-1.mid bach 0
1-pop-0.mid pop 1
1-pop-1.mid pop 1

Evaluation dataset

The evaluation data set contains 2000 MIDI files, each of which is single-melody music, with a tempo of 120 bpm, and each melody is 8 measures long. For the music generated by artificial intelligence, the melody source is the same as the development set, there are 1000 pieces. For human music, there are two sources of human music in the evaluation set, of which 90% of the music is obtained in the same way as the development set, and does not include the selected music in the development set. The other 10% of the music is obtained from the works that the composer has not yet published. This part of music does not exist in any currently known database. The MIDI files in the evaluation set are named in the following way. 

[style]-[id].mid

submission

All files should be packaged into a zip file for submission.

Official challenge submission consists of:

• System output file (*.csv)

• Technical report explaining in sufficient detail the method (*.pdf)

• A complete system code that can be run (in a separate folder)

System output should be presented as a single text-file (in CSV format, with a header row) containing classification result for each audio file in the evaluation set.

Detailed information for submission can be found on thesubmission page.

External data resources

Use of external data is allowed under the following conditions:

• The used external resource is clearly referenced and freely accessible to any other research group in the world. External data refers to public datasets or trained models. The data must be public and freely available before 15th of August 2020.

• Participants submit at least one system without external training data so that we can study the contribution of such resources. This condition applies only to cases where external audio datasets are used. In case of external data being pretrained models or embeddings, this condition does not apply.

• The list of external data sources used in training must be clearly indicated in the technical report.

• Participants inform the organizers in advance about such data sources, so that all competitors know about them and have equal opportunity to use them. Please send and email to the task coordinators; we will update the list of external datasets on the webpage accordingly. Once the evaluation set is published, the list of allowed external data resources is locked (no further external sources allowed).

Task rules

There are general rules for participants:

• Use of external data is allowed. See conditions for external data usage here。

• Manipulation of provided training and development data is allowed.

• Participants are not allowed to make subjective judgments of the evaluation data, nor to annotate it. The evaluation dataset cannot be used to train the submitted system.

• Classification decision must be done independently for each test sample.

Baseline system

The baseline system will be released at the same time when the development dataset is released.
