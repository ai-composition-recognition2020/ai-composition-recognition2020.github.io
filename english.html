AI composition
recognition competition

With the development
of artificial neural networks, many algorithms based on deep learning have been proposed and used in industry and academia for music generation. As a result, there are a series of coming legal issues such as the standards and procedures to identify the infringement of intellectual property rights. Exemplar cases include illegal application of AI composing algorithms, regarding intentional plagiarism as incident of AI composing algorithms, the accuracy and objectiveness of professional authentication and the dependence of litigation procedure on professional identification. As a preparation of such legal challenges, it is necessary to develop an objective way to evaluate music melodic similarity and describe music melodies. Such techniques can potentially enable digital forensic investigations for musical intellectual property issues.

Introduction

This data challenge aims to identify music melodies generated by artificial intelligence algorithms. The challenge provides a development dataset that contains melodies in two different music style and generated by a certain number of music generation algorithms.

There is no enrolment process in this data challenge. Any participants following the guidance in this page with success submission will be counted a team of participant. A development dataset will be released first for the development of algorithms with an evaluation dataset released one month later. The participant is expected to use the developed algorithms to label that how melodies in the evaluation dataset are generated (by either human or algorithms). Together with a copy of technical report on the developed algorithm, a CSV file containing labelling information is expected to be submitted via the submission system of Conference on Sound and Music Technology (CSMT) at the following page: (https://cmt3.research.microsoft.com/CSMT2020).

The technical report should follow the requirement of manuscript of CSMT (as a bilingual conference, both Chinese and English technical report is acceptable) and should be uploaded to arXiv before the deadline of challenge submission. When the result is submitted, please make sure the track of the system is selected as “Data Challenge”.

The organisation committee of the data challenge reserves the right that asks participants submitting source code for algorithm validation (an NDA could be signed if necessary). The participants would be disqualified if the participants refuse to submit source code upon request. The complete code should include: read the evaluation dataset function, main algorithms and generate system output function, and list the program package used in the code.

Participants are not allowed to make subjective judgments of the evaluation data, nor to annotate it. The participants would be disqualified if making subjective judgments

Development dataset

The development dataset contains 6000 MIDI files with monophonic melodies generated by artificial intelligence algorithms. The tempo is set to 120 bpm (beat per minute), and the length of each melody is 8 bars with incomplete phrase structures. There are two datasets with different music styles used as the training dataset of a certain number of algorithms, where the melodies in the development dataset are generated. The MIDI files in the development dataset are named in the following format.

development-id.mid

Evaluation dataset

The evaluation dataset contains 2000 MIDI files with exact configurations of development dataset with two exceptions: 1) A certain number of melodies composed by composers are added, some of which are published and some of which are composed for this competition. The music styles of these human composed melodies are the same styles of music in the training dataset as confirmed by musicologists. 2) There are a certain number of melodies generated by unmet algorithms that have minor changes based on the algorithms used in the development dataset. The MIDI files in the evaluation set are named in the following way.

evaluation-id.mid

Data release

Development dataset will be released on July 15, 2020.

External data resources

Use of external data is allowed under the following conditions:

• Only open source data that can be referenced is allowed. External data refers to public datasets or pre-trained models.

• The data must be public and freely available before 15th August 2020.

• The list of external data sources used in training must be properly cited and referenced in the technical report.

submission

All files should be packaged into a zip file for submission.

Official challenge submission consists of:

• System output file (*.csv)

• Technical report explaining in sufficient detail the method (*.pdf)

• A complete system code that can be run (in a separate folder)

System output should be presented as a single text-file (in CSV format, with a header row) containing classification result (float type) for each audio file in the evaluation set. and label of styles of music (str type).

                        file name               style（str）               Human（float）                  AI（float）

                        0.mid                   A                          0.8                            0.2
                        1.mid                   A                          0.75                           0.25
                        2.mid                   B                          0.6                            0.4
                        3.mid                   B                          0.1                            0.9

To avoid overlapping labels among all submitted systems, use following way to form your label:

[first name]_[last name]_[Abbreviation of institute of the corresponding author]

For example the systems would have the following labels:

• Hua_Li_BUPT

• Michael_Jordan_UNC

Make sure your zip-package follows provided file naming convention and directory structure. The zip package example can be downloaded here.

                        │   Hua_Li_BUPT_output.csv                     Task System output
                        │
                        └───Hua_Li_BUPT_code                           Task System code
                                  code.py
